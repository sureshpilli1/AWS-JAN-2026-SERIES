CIDR:

CIDR stands for:
Classless Inter-Domain Routing
CIDR is the modern way of defining networks and subnet sizes.
It replaced the old Class A, B, and C system because the class system wasted many IP addresses.

The masking system did not change. What changed was the way networks are allocated and represented.
Think of it this way:
Old (Classful Networking)
The network size was determined by the IP class.
Class	Default Mask
A	255.0.0.0 (/8)
B	255.255.0.0 (/16)
C	255.255.255.0 (/24)
Example:
192.168.1.10
People automatically assumed:
255.255.255.0
because it was a Class C address.
Problem
Suppose a company needs:
500 hosts
Class C:
254 hosts
Too small.
Class B:
65,534 hosts
Too large.
Huge waste of IP addresses.

Modern (CIDR)
CIDR removed the restriction that networks must be only:
/8
/16
/24

Now you can use:
/20
/21
/22
/23
/25
/26
/27
...

whatever size you need.
Example:
10.0.0.0/20
Mask:
255.255.240.0
This would have been impossible in the old classful system.


If the subnet mask concept stayed the same, how did networking change from Class A/B/C to CIDR?
The answer is:
Routers stopped assuming the mask based on the IP class and started carrying the mask information explicitly.

Old Classful Networking
Suppose you saw:
192.168.1.10
Router automatically thought:
192 = Class C
Class C = /24
Mask = 255.255.255.0
No need to specify the mask.
The router assumed it.

CIDR Idea
Instead of saying:
192.x.x.x = always /24
CIDR says:
Don't assume anything.
Always specify the prefix.
Examples:
192.168.1.0/23
192.168.1.0/25
192.168.1.0/27
All are valid.

What Changed in Routers?
Old router:
IP starts with 192
      ↓
Must be Class C
      ↓
Must be /24

CIDR router:
Network = 192.168.1.0/27
The router stores:
Network Address
+
Prefix Length
and no longer cares about classes.

Example
Classful world:
172.16.0.0
Automatically:
Class B
Mask = /16

CIDR world:
172.16.0.0/20
Mask:
255.255.240.0
Perfectly valid.
Even though it is not a Class B default mask.

What Changed?
Before
The mask was implied by the class.
10.x.x.x     → /8
172.x.x.x    → /16
192.x.x.x    → /24

After CIDR
The mask is explicitly specified.
10.0.0.0/22
10.0.0.0/19
10.0.0.0/27
No assumptions.

CIDR became possible because routers stopped deriving the subnet mask from the IP address class and instead started using an explicitly provided prefix length (such as /20 or /27). 
The subnet mask itself did not change; only the rule for determining it changed.

A network is a group of devices that share the same network portion of their IP addresses, as determined by the subnet mask. 
The network is identified by a network address (such as 192.168.1.0/24), and all hosts within that network can communicate directly without requiring a router.

Case 1: CPU Transistors
Suppose a transistor is inside the CPU.
When power is ON:
Voltage Present
The transistor can be:
ON  (1)
OFF (0)
When you shut down the computer:
Power OFF
There is no voltage.
Therefore:
All transistor states disappear
The CPU does not remember its previous ON/OFF states.

Case 2: RAM
RAM behaves similarly.
Suppose RAM contains:
A = 01000001
When power is removed:
RAM loses everything
This is why RAM is called:
Volatile Memory
because it forgets data when power is lost.

Case 3: SSD
An SSD is different.
It uses a special transistor called a:
Floating-Gate Transistor
or
Charge-Trap Transistor
How SSD Remembers Data
Imagine a tiny bucket.
Store 1
Electron Charge Stored
Result:

Store 0

Result:
0
The charge remains trapped even after power is removed.
So:
Power OFF
↓

↓

This is why SSDs are called:
Non-Volatile Storage

Case 4: HDD
Hard disks don't rely on transistor states for data storage.

Even after power is removed:

Why Does RAM Forget?
A RAM cell stores data using tiny electrical charges.
Example:
Capacitor Charged     → 1
Capacitor Discharged  → 0
Without power:
Charge leaks away
and the data disappears.

Why Does SSD Remember?
SSD cells trap electrons inside an insulated region.
The electrons cannot easily escape.
So:
Stored today
can remain stored for years without power.
